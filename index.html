Get Started

Get Started

HiGHS is a high performance serial and parallel solver for large-scale sparse linear programming (LP) problems developed in C++11. HiGHS is based on the dual revised simplex method implemented in HSOL, which was originally written by Qi Huangfu. Features such as presolve, crash and advanced basis start have been added by Julian Hall, Ivet Galabova. Other features, and interfaces to C, C#, FORTRAN, Julia and Python, have been written by Michael Feldmeier.

HiGHS is freely available under the MIT licence. Installing from source code requires CMake minimum version 3.0.0. HiGHS can be used as a standalone executable on Windows and Linux. There is a C++11 library which can be easily used within a C++ project.

Documentation can be built with doxygen. Download the latest revision of HiGHS from here and call ./doxygen in HiGHS/docs . Load HiGHS/docs/index.html into any web browser for descriptions of methods. Note, that we are currently working to improve our documentation. Your comments or specific questions would be greatly appreciated so please use the form below to submit any queries.

Guide

This guide sets out what HiGHS can be used for, and how to build and run it. Highs can be run from a command line or from within another application.

Specification

HiGHS can load LP and MIP models from either data files, or via data provided by another application. It can solve LP problems using the dual revised simplex method and MIP problems using branch-and-bound. The solution can be written to a file or retrieved directly for use within an application. When used within an application, HiGHS can be used to modify the current problem and re-solve it efficiently.

Building HiGHS

HiGHS uses CMake as a build system. First setup a build directory and call CMake as follows 

  mkdir build
  cd build
  cmake ..

Then compile the code using

  make

To perform a quick test to see whether the compilation was successful, run 

  ctest

Running HiGHS from the command line

In the following discussion, the name of the executable file generated is assumed to be highs. HiGHS can read plain text MPS files and LP files and the following command solves the model in ml.mps 

  highs ml.mps

HiGHS is controlled by means of options. Some may be specified directly on the command line. Many more are specified via a file. Formally, the usage is 

  highs [OPTION...] [file]

using the following options.

--model_file arg 
File of model to solve.
--presolve arg 
Presolve: "choose" by default - "on"/"off" are alternatives.
--solver arg 
Solver: "choose" by default - "simplex"/"ipm" are alternatives.
--parallel arg 
Parallel solve: "choose" by default - "on"/"off" are alternatives.
--time_limit arg 
Run time limit (double).
--options_file arg 
File containing HiGHS options.
-h, --help 
Print help.

A full specification of these options, and all options that are specified via a file, is given here. Within an options file, values are specified line-by-line as option_name = value. An example file containing default settings of all options is here.

Note that by setting the values of options solution_file, write_solution_to_file and write_solution_pretty appropriately, it is possible to write the solution to stdout (by setting solution_file = "") or to a file in either a pretty (human-readable) format or simple (computer-readable) format.

Running HiGHS from within an application

To run HiGHS from an application, it is necessary to load a model, solve it and extract the results. Many users will want to get and set option values. It is also possible to extract model data and modify a model. HiGHS can perform other operations for specialist applications.

Although HiGHS is written in C++, interfaces exist for C, C#, FORTRAN, Julia and Python. To within the limits of the language, they offer the same functionality that is available from C++, with method names that are either identical or distinguished by a consistent name extension. The discussion below refers to the C++ methods, followed by an account of the language-specific characteristics of the interfaces.

Example programs calling HiGHS from C, C#, FORTRAN, Julia and Python are in HiGHS/examples.

Before the C++ methods can be used, an instance of the Highs class must be created. Some methods are used to return data, and the others return an indication of success. In some cases this is a value of HighsStatus, and in others it is simply a boolean.

Load a model

The simplest way to use HiGHS to solve a problem is to load a model from a file using the method readModel(filename). Different file formats are recognised from the filename extension. HiGHS can read plain text MPS files and LP, but not compressed files. Alternatively, in C++, data generated by an application can be passed to HiGHS via an instance of the HighsLp class populated by the user and passed using the method passModel(HighsLp). In languages where such structures cannot be used, data constituting an LP problem can be passed via individual parameters.

Solve a model

HiGHS is used to solve a problem by calling run().

Extract the results

After solving a model, its status is the value returned by getModelStatus(). This value is of type HighsModelStatus, and may be interpreted via the names in the corresponding enum. The objective function value and iteration count are obtained using getObjectiveValue() and getIterationCount(). The solution and basis are returned by getSolution() and getSolution() respectively. Note that these are const references to internal data. HiGHS can also be used to write the solution to a file using writeSolution(filename), with the output going to stdout if filename is blank.

Extract model data

The numbers of column, rows and nonzeros in the model are returned by getNumCols(), getNumRows() and getNumEntries() respectively. Data for columns and rows from the model may be extracted using getCols and getRows, and specific matrix coefficients obtained using getCoeff.

Language interfaces

C# interface

From an application written in C#, HiGHS is run by creating a HighsLpSolver instance thus

  HighsLpSolver solver = new HighsLpSolver();

An LP problem may then be read in from a file solver.readModel(Filename), or passed to HiGHS by passing its component arrays of data to HighsModel. This returns an object that can then be passed to the HighsLpSolver instance via solver.passLp(*). The LP is solved using solver.run(), and data extracted as described above for the C++ interface.
